# خادم بروتوكول سياق النموذج (Dart and Flutter MCP Server Deep Dive)

يُعد **Dart and Flutter MCP server** أداة ثورية تربط بين أجهزة الذكاء الاصطناعي (AI Agents) وأدوات تطوير فلاتر ودارت. يرمز MCP إلى **Model Context Protocol**، وهو بروتوكول يسمح للذكاء الاصطناعي بفهم سياق الكود الخاص بك والقيام بأفعال حقيقية نيابة عنك.

---

## 🤔 لماذا نحتاج إلى خادم MCP؟

بدون هذا الخادم، يرى الذكاء الاصطناعي الكود كنص فقط. مع خادم MCP، يمكن للذكاء الاصطناعي:

* **فحص شجرة الودجت:** رؤية واجهة تطبيقك أثناء تشغيلها وتصحيح مشاكل التنسيق (Layout).
* **إدارة الاعتمادات:** البحث في `pub.dev` وإضافة حزم جديدة لملف `pubspec.yaml`.
* **التحكم في وقت التشغيل:** تنفيذ **Hot Reload** و **Hot Restart** لرؤية التغييرات فوراً.
* **إصلاح الأخطاء المعقدة:** تحليل الأخطاء الثابتة (Static) وأخطاء وقت التشغيل (Runtime) بسياق عميق.
* **تشغيل الاختبارات:** أتمتة عملية الاختبار وتحليل النتائج.

---

## 🛠️ كيف يعمل؟

يعمل خادم MCP كجسر تواصل. عندما تطلب من مساعد ذكي (مثل **Antigravity**) إصلاح خطأ، يقوم المساعد بإرسال طلب لخادم MCP، الذي بدوره يتحدث مع أدوات تطوير دارت (Dart Tooling Daemon) لجلب المعلومات المطلوبة أو تنفيذ أمر معين.

### الأدوات المتوافقة

* **Antigravity:** وكيل ذكي داخل المحرر يمكنه كتابة الكود وتشغيل الأوامر.
* **Gemini CLI:** أداة سطر أوامر لسير عمل الذكاء الاصطناعي.
* **Cursor & VS Code:** محررات تدعم بروتوكول MCP لتعزيز قدرات المساعد البرمجي.

---

## 🚀 البدء السريع

يتوفر خادم MCP كجزء من SDK دارت (بدءاً من إصدار 3.9). يمكن تشغيله عبر الأمر:

```bash
dart mcp-server
```

ثم يتم تكوينه داخل محررك المفضل (مثل ملف `mcp_config.json` في Antigravity أو `settings.json` في Gemini CLI).
