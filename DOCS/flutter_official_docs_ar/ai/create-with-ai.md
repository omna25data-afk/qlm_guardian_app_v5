# البناء باستخدام الذكاء الاصطناعي (Create with AI)

يوضح هذا الدليل كيف يمكنك توظيف أدوات الذكاء الاصطناعي لبناء ميزات ذكية لتطبيقات فلاتر الخاصة بك، بالإضافة إلى تحسين وتسريع عملية تطوير تطبيقات فلاتر ودارت بشكل عام.

---

## 📋 نظرة عامة

يمكن استخدام الذكاء الاصطناعي في فلاتر لغرضين رئيسيين:
1.  **بناء تطبيقات ذكية:** دمج ميزات مثل فهم اللغة الطبيعية وتوليد المحتوى مباشرة في تطبيقك باستخدام SDKs قوية مثل حزمة Firebase لـ Generative AI.
2.  **تسريع سير عمل التطوير:** استخدام أدوات مثل Gemini Code Assist و Gemini CLI للمساعدة في توليد الكود والبنية الأساسية للتطبيق.

تعتمد هذه الأدوات على **خادم Dart and Flutter MCP**، الذي يزود الذكاء الاصطناعي بسياق غني حول مشروعك.

---

## 🛠️ بناء تجارب ذكية مع فلاتر

يفتح استخدام الذكاء الاصطناعي في تطبيقك آفاقاً لتجارب مستخدم جديدة. للبدء، اطلع على المصادر التالية:

*   **Firebase AI Logic:** مكتبة Firebase الرسمية لاستخدام ميزات الذكاء الاصطناعي التوليدي مباشرة في فلاتر. متوافقة مع Gemini API و Vertex AI.
*   **Flutter AI Toolkit:** تطبيق عينة مع ودجت جاهزة (مثل `LlmChatView`) لمساعدتك في بناء ميزات الدردشة بسرعة.

---

## 💻 أدوات تطوير الذكاء الاصطناعي

الذكاء الاصطناعي ليس مجرد ميزة في التطبيق، بل هو مساعد قوي في دورة التطوير. الأدوات التالية تساعدك على كتابة الكود بشكل أسرع:

### Antigravity
وكيل ذكي داخل بيئة التطوير يمكنه قراءة وكتابة الكود، تشغيل أوامر الطرفية، والمساعدة في بناء ميزات معقدة. يتميز بقدرات "وكيلة" تسمح له باتخاذ قرارات وتنفيذ مهام متعددة الخطوات.

### Gemini Code Assist
متعاون ذكي متوفر لـ VS Code و Android Studio. لديه فهم عميق لمشروعك ويساعد في:
*   إكمال وتوليد الكود برمجياً.
*   الدردشة داخل المحرر لطرح أسئلة تقنية.
*   تصحيح الأخطاء وتقديم شروحات للكود.

### Gemini CLI
أداة سطر أوامر لسير عمل الذكاء الاصطناعي، تسمح لك بالتفاعل مع نماذج Gemini دون مغادرة الطرفية. يمكنك استخدامها لإنشاء الودجت بسرعة أو أتمتة مهام Git.

## 🌐 خادم بروتوكول سياق النموذج (MCP Server)

لكي يقدم الذكاء الاصطناعي مساعدة حقيقية، يحتاج للتواصل مع أدوات مطوري فلاتر. يقوم خادم MCP بهذا الدور، حيث يسمح للذكاء الاصطناعي بـ:
*   **فحص شجرة الودجت:** تشخيص مشاكل التنسيق بصرياً.
*   **إدارة الاعتمادات:** البحث في `pub.dev` وإضافة الحزم.
*   **التحكم في وقت التشغيل:** تنفيذ Hot Reload لرؤية التغييرات فوراً.

للتعمق، راجع دليل [خادم MCP](mcp-server.md).
