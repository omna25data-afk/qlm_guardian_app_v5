# المدخلات والمخرجات الهيكلية (Structured Input & Output)

لضمان عمل برامجك بشكل سليم مع الذكاء الاصطناعي، يجب أن يكون التواصل واضحاً وغير غامض. يتم ذلك عبر استخدام البيانات المنظمة (Structured Data) في كلا الاتجاهين.

---

## 📥 المدخلات الهيكلية (Structured Input)

يمكن للذكاء الاصطناعي استقبال أي نص تقريباً، لكن النتائج تتحسن بشكل كبير عند استخدام تنسيقات منظمة مثل **JSON** أو **CSV** أو **Markdown**.
بالإضافة للنصوص، تدعم نماذج Gemini البيانات الثنائية (Binary) مثل الصور وملفات PDF.

**مثال: إرسال صورة مع طلب (Vision):**
في فلاتر، نقوم بتحويل الصورة إلى أجزاء (Parts) وإرسالها مع نص الأمر في طلب واحد متكامل.

---

## 📤 المخرجات الهيكلية (Structured Output)

تجد النماذج اللغوية أحياناً صعوبة في الالتزام بتنسيق معين (مثل JSON) إذا لم يُحدد بدقة. لضمان الحصول على JSON قابل للتحليل برمجياً:

### 1. تحديد المخطط (Schema)
استخدم كائن `Schema` عند تهيئة النموذج لتحديد الحقول المطلوبة وأنواع بياناتها (String, Integer, Array, etc.).

### 2. تعليمات النظام (System Instruction)
أفضل النتائج تأتي عندما تذكر مخطظ الـ JSON المطلوب (JSON Schema) داخل تعليمات النظام بوضوح.

```dart
// مثال لتعريفSchema في فلاتر
static final _mySchema = Schema(
  SchemaType.object,
  properties: {
    'answer': Schema(SchemaType.string),
    'confidence': Schema(SchemaType.number),
  },
);
```

---

## ⚖️ الموازنة بين المرونة والدقة

المخرجات الهيكلية هي ما يجعل من الممكن دمج الذكاء الاصطناعي في منطق تطبيقك. حتى لو كانت البيانات المنتجة من الذكاء الاصطناعي غير صحيحة أحياناً، فإن وجودها بتنسيق JSON يسمح لتطبيقك بمعالجتها، اكتشاف الخطأ فيها، أو حتى إظهارها للمستخدم ليصححها.
