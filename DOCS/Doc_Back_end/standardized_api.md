# توثيق الربط البرمجي الموحد (Standardized API Integration)

هذه الوثيقة تلخص "العقد" الموحد بين الباكند وتطبيق الجوال (فلاتر) لضمان استقرار الربط وسهولة البرمجة.

## 1. الهيكل العام للاستجابة (Response Structure)

جميع الردود من السيرفر تأتي بتنسيق JSON موحد:

```json
{
  "status": true,
  "code": 200,
  "message": "نص الرسالة للمستخدم",
  "data": { ... }
}
```

* **status:** (Boolean) يحدد نجاح أو فشل الطلب منطقياً.
* **code:** (Integer) كود HTTP (مثل 200, 201, 401, 422).
* **message:** نص عربي جاهز للعرض مباشرة للمستخدم.
* **data:** الحاوية الرئيسية للبيانات (Array أو Object).

## 2. مجموعة Postman (Postman Collection)

لتبسيط عملية الاختبار، تم إنشاء مجموعة Postman جاهزة تحتوي على أهم المسارات مع أمثلة حية.

* [تحميل ملف Postman Collection](./postman_collection.json)

## 3. ملاحظات هامة

* **المصادقة:** يتم استخدام `Bearer Token` عبر Laravel Sanctum. صلاحية التوكن مضبوطة على **7 أيام**.
* **الأخطاء:** في حالة أخطاء التحقق (Validation)، تأتي الأخطاء في حقل `errors` داخل جسم الاستجابة.
* **التواريخ:** جميع التواريخ ترسل بصيغة `ISO 8601`.

للمزيد من التفاصيل التقنية حول أفضل الممارسات، يرجى مراجعة [دليل أفضل الممارسات](file:///F:/herd/custom_filment/system_endpoints/api_best_practices.md).
