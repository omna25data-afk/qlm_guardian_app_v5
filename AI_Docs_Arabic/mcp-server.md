يناقش هذا الدليل خادم MCP (بروتوكول سياق النموذج - Model Context Protocol) الخاص بـ Dart و Flutter.

:::experimental
يُعد خادم MCP لـ Dart و Flutter تجريبياً وعرضة للتطور السريع. تتطلب التعليمات التالية إصدار Dart 3.9 أو أحدث.
:::

## نظرة عامة (Overview)

يقوم خادم MCP الخاص بـ Dart و Flutter بإتاحة إجراءات أدوات تطوير Dart و Flutter لعملاء المساعد الذكي المتوافقين. MCP هو بروتوكول يتيح الاتصال بين أدوات التطوير ومساعدي الذكاء الاصطناعي، مما يسمح للمساعد بفهم سياق الكود وتنفيذ إجراءات نيابة عن المطور.

يمكن لخادم MCP هذا العمل مع أي عميل MCP يدعم الإدخال/الإخراج القياسي (stdio) كوسيلة نقل. وللوصول إلى كافة الميزات، يجب أن يدعم العميل "أدوات" (Tools) و "موارد" (Resources)، ولأفضل تجربة يجب أن يدعم "الجذور" (Roots).

تتضمن بعض ميزات الخادم:

* تحليل وإصلاح الأخطاء في كود المشروع.
* التفاعل مع التطبيق أثناء التشغيل (Introspect running application).
* البحث في موقع pub.dev عن الحزم المناسبة.
* إدارة تبعيات الحزم في ملف `pubspec.yaml`.
* تشغيل الاختبارات وتحليل نتائجها.
* تنسيق الكود باستخدام إعدادات تنسيق Dart.

## إعداد عميل MCP الخاص بك (Set up your MCP client)

يوضح هذا القسم كيفية تشغيل الخادم باستخدام أمر `dart mcp-server` وإعداده في أدوات ومحررات شائعة:

### Antigravity

يمكن إضافة الخادم إما من خلال متجر MCP المدمج بالبحث عن "Dart" وتثبيته، أو يدوياً عن طريق تعديل ملف `mcp_config.json` وإضافة إعدادات `dart-mcp-server` التي تنفذ أمر `dart mcp-server`.
يُنصح أيضاً بتثبيت إضافات Dart و Flutter.

### Gemini CLI

يتم تكوين الإضافة عن طريق إضافة إعداد Dart إلى قسم `mcpServers` في ملف `settings.json` (إما على مستوى الجهاز أو مستوى المشروع).

### بيئات أخرى

يمتد الشرح ليشمل تعليمات التكوين في بيئات أخرى مثل:

* Firebase Studio
* Gemini Code Assist في VS Code
* GitHub Copilot في VS Code
* Cursor
* OpenCode
* Claude Code
* Codex CLI

## استخدام عميل MCP الخاص بك (Use your MCP client)

بعد إعداد الخادم، يتمكن نموذج اللغة الكبير (LLM) من تقرير متى وأي الأدوات يجب استخدامها بناءً على طلبك باللغة الطبيعية. يستعرض الدليل مثالين:

1. **إصلاح خطأ واجهة المستخدم (Runtime layout error):**
   عند طلب إصلاح مشاكل في التصميم، يستخدم المساعد الذكي الخادم لقراءة أخطاء التشغيل، فحص شجرة الويدجت (Widget tree)، وتطبيق حل مناسب للمشكلة.

2. **إضافة وظيفة جديدة عبر البحث عن حزم (Package Search):**
   عند طلب إضافة رسم بياني، سيقوم المساعد بالبحث عن أفضل حزمة في pub.dev، إضافتها إلى التبعيات في `pubspec.yaml`، وإنشاء كود مبدئي متكامل.
